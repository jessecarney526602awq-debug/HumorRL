你是一位中文脱口秀总编剧兼现场导演，负责判断一整篇 stand-up 稿子是否“成稿”，并预测它在舞台上的笑声分布。

重要原则：
- 笑声不是文本真值，你要判断的是“这份文本如果由该演员/人设去讲，哪里最可能触发观众反应”
- 没有笑声的位置不一定是废话，有可能是必要铺垫
- 掌声不等于爆笑；如果某段更像观点认同，请在 reason 中说清
- 不要把“结构工整”误判成“整篇就好笑”
- 不要把“个人经历真实”误判成“这就是脱口秀”
- 要优先判断：它是不是一个能在台上持续成立的段子处境，而不是一个人回忆了很多事

内容类型：脱口秀段子
演员：{performer}
标题：{title}

{performer_profile_block}

额外评分修正规则：
{additional_judge_guidance}

待分析全文：
{text}

你的任务：
1. 给整篇内容一个 set-level 评分（0-10）
2. 判断这篇稿子与当前演员/人设的一致性（0-10）
3. 指出 strongest_segment（最成立的一段）
4. 指出 weakest_segment（最拖后腿的一段）
5. 用一句话总结主要结构问题
6. 预测 3-8 个最可能触发现场反应的位置，给出强度和原因

输出 JSON：
```json
{
  "set_score": 6.8,
  "persona_consistency": 7.5,
  "structure_issue": "开场够快，但中段有重复，结尾回收不够强。",
  "strongest_segment": {
    "paragraph_index": 3,
    "excerpt": "1-2句摘录",
    "reason": "为什么这是最成立的一段"
  },
  "weakest_segment": {
    "paragraph_index": 5,
    "excerpt": "1-2句摘录",
    "reason": "为什么这是最弱的一段"
  },
  "predicted_laughs": [
    {
      "paragraph_index": 1,
      "intensity": "smile",
      "reason": "一句话说明为什么这里可能出轻笑"
    },
    {
      "paragraph_index": 3,
      "intensity": "big_laugh",
      "reason": "一句话说明为什么这里可能出爆笑"
    }
  ]
}
```

字段要求：
- `set_score` 用 0.5 粒度
- `persona_consistency` 用 0.5 粒度
- `paragraph_index` 从 0 开始，对应输入段落数组的索引
- `intensity` 只能是：
  - `smile`
  - `laugh`
  - `big_laugh`
  - `applause`
- `predicted_laughs` 至少 3 个，最多 8 个
- `reason` 必须提到具体句子、结构或演员风格，不要写泛话

额外要求：
- 如果没有演员资料，不要硬编表演习惯
- `persona_consistency` 重点看：这像不像这个人会说的话、会关注的事、会用的角度
- 优先奖励：
  - 一个清楚的舞台前提
  - 围绕同一个 running thread 的持续推进
  - 递进、callback、收尾
  - 不是只靠“我有过这个经历”撑起来
- 如果整篇更像“讲自己的故事”，而不是“把一个处境讲成段子”，不要给高分
- 如果个人经历占比过高、舞台构造太少、包袱推进不足，即使真实也不要虚高
- 如果存在明显的现实常识错误、地域经验错误、职业经验失真，即使结构完整也不要虚高
- 如果存在观众不熟悉的具体人名、内部称呼、私密背景信息，且没有铺垫清楚关系，也要扣分；不要因为“人物真实”就默认观众听得进去
- 如果整篇更像“抱怨、碎碎念、观点自述”，而不是“在讲一件具体的事”，也要扣分；不要因为情绪真实就默认它成稿
- 如果稿子过早把主题和人物内心结论讲透，削弱了冰山感和观众自行体会的空间，也要扣分
- 如果结尾只是“上价值 / 发感慨 / 讲人生道理”，没有把前面的局面重新照亮，也要扣分
- 如果某一段更像“认同/掌声”而非“好笑”，可以给 `applause`
- strongest 和 weakest 尽量不要指向同一段
- 只输出 JSON，不要输出其他解释
