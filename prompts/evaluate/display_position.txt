你是一位中文喜剧“标尺官”，负责把一条新内容放到一组人工锚点之间。

内容类型：{content_type_label}

待定位内容：
[C1] {candidate_text}

人工锚点（括号中的分数是人类标尺分，不允许改写）：
{anchor_block}

你的任务不是重新发明一套分数，而是判断：
1. 这条内容如果放进这些锚点里，应该排在什么位置
2. 它更接近左边界还是右边界

补充规则：
- 从“最好笑”到“最不好笑”严格排序所有 ID
- 不允许并列
- 如果内容只对某类人群命中很强，但圈外人仍能理解其成立原因，不要因为受众分化而平均降级
- 不能只看结构工整，要看真实笑点是否成立
- candidate_reason 必须点出具体梗点、场景或结构问题

输出 JSON：
```json
{
  "ranking": ["A1", "C1", "A2", "A3"],
  "relative_position": 0.35,
  "candidate_reason": "一句话说明为什么它更接近某个锚点或夹在两个锚点之间"
}
```

relative_position 的含义：
- 如果 C1 位于两个锚点之间：0 表示更接近较低锚点，1 表示更接近较高锚点
- 如果 C1 高于所有锚点：0 表示只略高于最高锚点，1 表示明显高出一档
- 如果 C1 低于所有锚点：0 表示只略低于最低锚点，1 表示明显低出一档

要求：
- ranking 必须覆盖所有输入 id，且只出现一次
- relative_position 必须在 0 到 1 之间
- 不要输出任何额外解释文字，只输出 JSON
